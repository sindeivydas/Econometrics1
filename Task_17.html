Galima pastebėti, kad kiekvienu atveju duomenys pasiskirstę aplink įsivaizduojamą tiesę, todėl sukuriami tiesinės regresijos modeliai kiekvienam variantui:

reg11 = lm(y.t1c1~x.t1c1)
reg12 = lm(y.t1c2~x.t1c2)
reg21 = lm(y.t2c1~x.t2c1)
reg22 = lm(y.t2c2~x.t2c2)

Toliau pagal tuos pačius kriterijus išskirstomi input duomenys:

x11 <- input[(input$color == 1) & (input$type == 1), ("x")]
x12 <- input[(input$color == 2) & (input$type == 1), ("x")]
x21 <- input[(input$color == 1) & (input$type == 2), ("x")]
x22 <- input[(input$color == 2) & (input$type == 2), ("x")]

Turimiems x prognozuojamos y reikšmės, naudojant tiesinės regresijos modelius atitinkamiems duomenų variantams:

prog11 = predict(reg11, newdata=data.frame(x.t1c1 = x11))
prog12 = predict(reg12, newdata=data.frame(x.t1c2 = x12))
prog21 = predict(reg21, newdata=data.frame(x.t2c1 = x21))
prog22 = predict(reg22, newdata=data.frame(x.t2c2 = x22))

Šios prognozės įrašomos į naują kintamąjį y taip, kad atitiktų x reikšmes paeiliui, ir prijungiamos prie duomenų masyvo input:

dim(input) #patikrinamas eilučių skaičius
## [1] 2000    4
y <- replicate(2000, 0) 
y[(input$color == 1) & (input$type == 1)] = prog11
y[(input$color == 2) & (input$type == 1)] = prog12
y[(input$color == 1) & (input$type == 2)] = prog21
y[(input$color == 2) & (input$type == 2)] = prog22
